1.When you need to use abstract class then interface?

- We use abstract classes and methods when we need to pass the child classes to specific methods
that they inherit from the parent class, but we can't commit to the code that should be written in the methods.
- With interfaces it is possible to specify which methods a class should implement.
Interfaces make it easy to use a variety of different classes in the same way.

2. What is the difference of abstract class and interface?

1. abstract classes have properties whereas interfaces cannot have them.
2.All interface methods must be public while abstract class methods are public or protected.
3.All methods in an interface are abstract, so they cannot be implemented in code and the abstract keyword is not necessary
an abstraction cannot contain actual code, but it can simply define a method "signature".
4. Classes can implement an interface and at the same time inherit from another class.


3. Write an example of abstract class and how to implement it!

// Parent class
abstract class Flowers{
public $name_despo;;
public function __construct($name_despo;){
$this->name = $name_despo;;
}
abstract public function intro() : string;
}

// Child classes
class Tulip extends Flowers{
public function intro() : string{
return "$this->name have white color!";
}
}

class Jasmin extends Flowers {
public function intro() : string {
return "$this->name is multicoloured!";
}
}

// Create objects from the child classes
$tulip = new tulip("Tulip");
echo $tulip->intro();
echo "br";

$jasmin = new jasmin("Jasmin");
echo $jasmin->intro();
echo "br";
?>

Output:
Tulip is multicoloured!
Jasmin have white color!
